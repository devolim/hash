Erste Schritte

Laden Sie Hash kostenlos aus dem App Store (iOS) oder Google Play (Android) herunter. Die Anwendung ist kostenlos und wird es immer bleiben. Keine In-App-Käufe, kein Abonnement.
Nein. Hash fragt nach keiner Telefonnummer, keiner E-Mail-Adresse und keinem Passwort. Beim ersten Start erstellen Sie einen lokalen PIN-Code und eine anonyme Hash-ID wird automatisch generiert. Das ist alles.
Ihre Hash-ID ist eine einzigartige und anonyme Kennung, die Ihrem Gerät zugewiesen wird. Sie ist mit keinerlei persönlichen Informationen verknüpft. Niemand kann anhand Ihrer Hash-ID auf Sie zurückverfolgen.
Zwei Methoden stehen zur Verfügung:

1. QR-Code (empfohlen) — Einer von Ihnen zeigt seinen QR-Code in der App an, der andere scannt ihn. Die Verbindung erfolgt sofort und anonym.

2. Temporärer Code — Hash generiert einen 6-stelligen Code, der 5 Minuten gültig ist. Teilen Sie ihn mit Ihrem Kontakt, der ihn dann in der App eingibt, um Sie hinzuzufügen.
Hash funktioniert nach dem Prinzip „ein Gerät = ein Konto". Dies stärkt Ihre Sicherheit: Ihre Verschlüsselungsschlüssel existieren nur auf einem einzigen Gerät. Sie können Ihr Konto jedoch über die Einstellungen auf ein neues Gerät übertragen.

Nachrichten

Ein flüchtiges Medium ist ein Foto oder ein Video, das Ihr Empfänger nur einmal ansehen kann. Sobald es angesehen wurde, verschwindet es endgültig. Es ist unmöglich, es erneut anzusehen oder zu speichern.
Sie können Fotos, Videos, Dokumente, Sprachnachrichten und Dateien jeder Art senden. Alle Dateien werden vor dem Versand mit AES-256-GCM verschlüsselt und nach dem Empfang vom Server gelöscht.
Hash verwendet das Signal-Protokoll für die Ende-zu-Ende-Verschlüsselung Ihrer Nachrichten. Jede Nachricht wird auf Ihrem Gerät verschlüsselt, bevor sie gesendet wird. Der Server sieht nur einen unlesbaren Datenblock. Nur Ihr Empfänger besitzt den Schlüssel zum Entschlüsseln. Darüber hinaus stellt das Sealed-Sender-Protokoll sicher, dass selbst der Server nicht weiß, wer was an wen sendet.

Anrufe

Ja. Alle Audio- und Videoanrufe sind Ende-zu-Ende-verschlüsselt. Die Verbindung erfolgt direkt zwischen den Geräten (Peer-to-Peer). Es werden keine Aufzeichnungen gemacht. Keine Dauer, kein Anrufverlauf wird gespeichert.
Die Qualität der Anrufe hängt von Ihrer Internetverbindung ab. Für ein optimales Erlebnis:

• Verwenden Sie eine stabile WLAN-Verbindung
• Stellen Sie sicher, dass Ihre 4G/5G-Verbindung ausreichend ist
• Schließen Sie andere Anwendungen, die Bandbreite verbrauchen
• Vergewissern Sie sich, dass beide Teilnehmer eine gute Verbindung haben
Ja. Hash unterstützt Audio- und Videoanrufe. Beide sind Ende-zu-Ende-verschlüsselt. Sie können während eines Anrufs zwischen Audio und Video wechseln.

Sicherheit

Der Vash-Modus ist eine einzigartige Funktion von Hash. Sie richten einen zweiten PIN-Code ein. Wenn jemand Sie zwingt, die Anwendung zu öffnen, geben Sie diesen zweiten Code ein. Die App öffnet sich normal, aber alle Ihre Daten werden lautlos und unwiderruflich gelöscht — Kontakte, Nachrichten, Gespräche, Dateien. Der Beobachter sieht eine leere Anwendung. Es wird keine Warnung angezeigt.

Achtung: Diese Aktion ist unwiderruflich. Alle Daten gehen endgültig verloren.
Der Panikknopf ermöglicht es, alle Anwendungsdaten mit einer einzigen Aktion sofort und endgültig zu löschen. Im Gegensatz zum Vash-Modus ist diese Löschung sichtbar — der Benutzer bestätigt explizit die Vernichtung seiner Daten.
Die Schüttel-Sperre ermöglicht es, die Anwendung sofort zu sperren, indem Sie Ihr Telefon schütteln. Nützlich, wenn jemand auf Sie zukommt, während Sie Hash verwenden. Sie können die Erkennungsempfindlichkeit in den Einstellungen anpassen.
Sealed Sender ist ein Protokoll, das verhindert, dass der Server weiß, wer mit wem kommuniziert. Ihre Nachrichten werden völlig anonym übertragen: Absender, Empfänger und Inhalt sind für unsere Infrastruktur unsichtbar. Selbst wenn jemand den Datenverkehr abfängt, kann er nicht feststellen, wer eine Nachricht an wen sendet.
Konzeptbedingt speichert Hash keinerlei Informationen auf unseren Servern, die das Zurücksetzen Ihres PINs ermöglichen würden. Wenn Sie einen Wiederherstellungssatz eingerichtet haben, können Sie diesen verwenden, um wieder Zugang zu erhalten. Andernfalls sind die Anwendungsdaten unwiederbringlich verloren und Sie müssen Hash neu installieren, um von vorne anzufangen. Dies ist eine Sicherheitsentscheidung, kein Fehler.
Nein. Ende-zu-Ende-Verschlüsselung bedeutet, dass nur Sie und Ihr Empfänger Ihre Nachrichten lesen können. Der Server sieht nur verschlüsselte Daten, die er technisch nicht entschlüsseln kann. Darüber hinaus verhindert das Sealed-Sender-Protokoll, dass der Server weiß, wer mit wem kommuniziert. Selbst wenn wir eine gerichtliche Anordnung erhielten, wären wir nicht in der Lage, den Inhalt Ihrer Gespräche bereitzustellen.

Benachrichtigungen

Gehen Sie zu Einstellungen > Benachrichtigungen und ändern Sie den angezeigten Namen. Sie können „Erinnerung", „Notizen" oder einen beliebigen Namen eingeben. Die Benachrichtigungen zeigen dann diesen Namen statt „Hash" an, wodurch die Anwendung im Alltag unsichtbar wird.
Aktivieren Sie unter Einstellungen > Benachrichtigungen die Option zum Verbergen des Inhalts. Die Benachrichtigungen zeigen dann eine generische Meldung anstelle des tatsächlichen Inhalts Ihrer Nachrichten an. Niemand kann Ihre Gespräche über Ihre Schulter lesen.
Überprüfen Sie folgende Punkte:

• Berechtigungen — Stellen Sie sicher, dass Hash die Erlaubnis hat, Benachrichtigungen zu senden (Telefoneinstellungen > Hash > Benachrichtigungen)
• Nicht stören-Modus — Überprüfen Sie, ob dieser nicht aktiviert ist
• Akku — Einige Telefone deaktivieren Benachrichtigungen im Energiesparmodus. Schließen Sie Hash von der Akkuoptimierung aus
• Verbindung — Überprüfen Sie Ihre Internetverbindung

Konto

Gehen Sie zu Einstellungen > Konto > Mein Konto übertragen. Die Anwendung generiert einen Übertragungscode. Geben Sie diesen Code auf Ihrem neuen Gerät ein. Die Übertragung verschiebt Ihre Hash-ID und Ihre Kontakte auf das neue Gerät. Das alte Gerät wird automatisch abgemeldet.

Hinweis: Nachrichten werden nicht übertragen, entsprechend dem Prinzip flüchtiger Nachrichten von Hash.
Gehen Sie zu Einstellungen > Konto > Mein Konto löschen. Die Löschung erfolgt sofort und ist unwiderruflich. Alle Ihre Daten (Hash-ID, Kontakte, Verschlüsselungsschlüssel, Nachrichten) werden endgültig von Ihrem Gerät und unseren Servern gelöscht.

Sie können die Löschung auch per E-Mail beantragen: hash-support@devolim.fr

Fehlerbehebung

Überprüfen Sie Ihre Internetverbindung. Nachrichten werden gesendet, sobald die Verbindung wiederhergestellt ist. Wenn das Problem weiterhin besteht, versuchen Sie, die Anwendung zu schließen und erneut zu öffnen. Wenn Ihr Kontakt sein Konto gelöscht hat oder Sie blockiert hat, kann die Nachricht nicht zugestellt werden.
Versuchen Sie folgende Schritte:

• Stellen Sie sicher, dass Sie die neueste Version von Hash haben
• Starten Sie Ihr Telefon neu
• Überprüfen Sie, ob Ihr Betriebssystem auf dem neuesten Stand ist
• Wenn das Problem weiterhin besteht, kontaktieren Sie uns unter hash-support@devolim.fr mit einer Beschreibung des Problems
Senden Sie eine E-Mail an hash-support@devolim.fr mit einer Beschreibung des Problems, Ihrem Gerätemodell und der Version Ihres Betriebssystems. Screenshots oder eine detaillierte Beschreibung der Schritte zur Reproduktion des Fehlers sind sehr hilfreich.
