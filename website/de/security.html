Sicherheit

Keine Spuren.
Konzeptbedingt, nicht nur versprochen.

Wir haben nicht einfach Verschlüsselung zu einem herkömmlichen Messenger hinzugefügt. Wir haben jede Schicht so aufgebaut, dass es physisch unmöglich ist, auf Ihre Daten zuzugreifen.

DER WEG EINER NACHRICHT

VON IHREM FINGER ZUM BILDSCHIRM
IHRES KONTAKTS.

Verfolgen Sie die Reise einer Hash-Nachricht, Schritt für Schritt. Sie werden verstehen, warum niemand — nicht einmal wir — Ihre Gespräche lesen kann.

Links sehen Sie, was Sie sehen: ein einfaches "Hallo". Aber was der Server empfängt, ist eine Folge von 610 Bytes völlig unlesbarer verschlüsselter Daten.

Zwei Verschlüsselungsschichten. Ein einzigartiger flüchtiger Schlüssel für jede Nachricht. Selbst wenn jemand alles abfängt — es ist nutzlos ohne den privaten Schlüssel des Empfängers, der nur auf dessen Telefon gespeichert ist.

1

Sie tippen Ihre Nachricht

Sie schreiben "Hallo!" in der App. Zu diesem Zeitpunkt existiert die Nachricht nur im Speicher Ihres Telefons. Noch wurde nichts gesendet.

2

Verschlüsselung mit dem Signal Protocol

Sobald Sie auf "Senden" drücken, wird Ihre Nachricht durch das Signal Protocol in eine unverständliche Zeichenfolge umgewandelt.

Einfach gesagt: Ihre Nachricht wird in einen digitalen Tresor eingeschlossen, zu dem nur Ihr Kontakt den Schlüssel besitzt. Selbst wenn jemand die Nachricht unterwegs abfängt, sieht er nur eine sinnlose Folge von Buchstaben und Zahlen.

DOUBLE RATCHET CURVE25519 AES-256
3

Sealed Sender: der unsichtbare Umschlag

Stellen Sie sich vor, Sie senden einen Brief per Post, aber ohne Absenderadresse auf dem Umschlag. Genau das macht das "Sealed Sender"-System von Hash.

Die verschlüsselte Nachricht wird in einem anonymen Postfach auf unserem Server hinterlegt. Der Server weiß nicht, wer die Nachricht hinterlegt hat. Er weiß nur, in welches Postfach er sie legen soll — aber er kennt nicht die Identität des Postfachbesitzers.

4

Durchleitung über den Server

Der Server spielt die Rolle eines blinden Briefträgers. Hier ist, was er sieht:

Was der Server weiß: "Jemand hat etwas in Postfach #A7X9 hinterlegt."
Was der Server NICHT weiß: Wer es gesendet hat. Was es ist. Für wen es wirklich bestimmt ist.

Der Server speichert keine Metadaten: keine IP-Adresse, keinen Sendezeitstempel, keine Verbindung zwischen Absender und Empfänger.

5

Ihr Kontakt empfängt die Nachricht

Das Telefon Ihres Kontakts ruft den Inhalt seines anonymen Postfachs ab. Es entschlüsselt die Nachricht lokal mit seinem privaten Schlüssel — der niemals sein Gerät verlassen hat.

"Hallo!" erscheint auf seinem Bildschirm. Mission erfüllt.

6

Die Nachricht verschwindet vom Server

Sobald Ihr Kontakt den Empfang bestätigt, löscht der Server die Nachricht endgültig. Kein Papierkorb, kein Archiv, kein Backup. Die Daten werden aus der Datenbank gelöscht.

7

Lokales Verfallsdatum

Auf dem Gerät Ihres Kontakts zerstört sich die Nachricht selbst gemäß der von Ihnen gewählten Dauer: sofort nach dem Lesen, 5 Minuten, 1 Stunde... Sie entscheiden.

Am Ende: Keine Spur auf dem Server, keine Spur auf den Geräten. Die Nachricht existierte nur so lange, wie sie gelesen wurde, dann ist sie für immer verschwunden.

Architektur

5 SCHUTZSCHICHTEN.

Hash basiert nicht auf einer einzigen Technologie. Jede Schicht verstärkt die anderen, wodurch ein System entsteht, bei dem selbst die Kompromittierung eines Elements Ihre Daten nicht gefährdet.

1

ENDE-ZU-ENDE-VERSCHLÜSSELUNG (SIGNAL PROTOCOL)

Jede Nachricht wird dank des "Double Ratchet"-Algorithmus mit einem einzigartigen Schlüssel verschlüsselt. Einfach gesagt: Selbst wenn jemand eine Nachricht entschlüsselt, kann er die nächste nicht entschlüsseln. Jede Nachricht hat ihr eigenes Schloss.

Für Dateien (Fotos, Videos, Dokumente) verwendet Hash eine zusätzliche AES-256-GCM-Verschlüsselung. Die Datei wird verschlüsselt, bevor sie Ihr Telefon verlässt.

2

SEALED SENDER (NETZWERK-ANONYMITÄT)

Herkömmliche Messenger senden Ihre Nachrichten mit Ihrer angehängten Identität — das ist wie Ihren Namen auf den Umschlag zu schreiben. Hash verwendet ein System von "anonymen Postfächern": Der Server hinterlegt die Nachricht, ohne zu wissen, wer sie gesendet hat.

Ergebnis: Selbst wenn der Server kompromittiert wird, kann ein Angreifer nicht rekonstruieren, wer mit wem spricht.

3

AUTOMATISCHE LÖSCHUNG

Nachrichten werden nach bestätigtem Empfang vom Server gelöscht. Selbst wenn eine Nachricht nie abgeholt wird, wird sie nach 24 Stunden automatisch zerstört.

Auf Ihrem Telefon zerstören sich Nachrichten selbst gemäß der von Ihnen gewählten Dauer: sofort, 5 Min., 15 Min., 30 Min., 1 Std., 3 Std., 6 Std. oder 12 Std.

4

LOKALER ZUGRIFFSSCHUTZ

Die App wird durch einen 6-stelligen PIN-Code und/oder Biometrie (Face ID, Fingerabdruck) geschützt. Nach zu vielen Fehlversuchen sperrt sich die App mit einer exponentiellen Verzögerung.

5

GESICHERTE DATENBANK

Serverseitig kann kein Benutzer direkt in die Datenbank schreiben. Alle Operationen (Nachricht senden, Kontakt hinzufügen, Konto löschen) laufen über gesicherte Funktionen, die jede Aktion überprüfen, bevor sie ausgeführt wird.

Es ist wie ein Bankschalter: Sie berühren nie selbst den Tresor. Sie stellen eine Anfrage, und das System prüft, ob Sie berechtigt sind, bevor es handelt.

VASH-MODUS.
WELTWEIT EINZIGARTIG.

EIN NOTFALL-SICHERHEITSSYSTEM, DAS IN KEINER ANDEREN MESSENGER-APP DER WELT EXISTIERT.

Hash ist der einzige Messenger, der einen Zerstörungscode bietet. Ein zweiter PIN-Code, der die App scheinbar normal öffnet — aber in Wirklichkeit stillschweigend alle Ihre Daten löscht im Hintergrund.

Gespräche, Kontakte, Verschlüsselungsschlüssel, Dateien — alles wird in Sekundenbruchteilen zerstört. Der Beobachter sieht eine leere App und denkt, Sie benutzen sie nicht. Keine Spur. Keine Warnung. Unwiderruflich.

Signal hat diese Funktion abgelehnt. Telegram hat sie nie in Betracht gezogen. WhatsApp hat nicht einmal daran gedacht. Nur Hash hat sie implementiert.

Das Szenario

Stellen Sie sich vor: Jemand verlangt, dass Sie Ihr Telefon entsperren. Eine Kontrolle, eine Drucksituation, eine böswillige Person. Sie können nicht ablehnen. Was tun?

Die Lösung: Zwei PIN-Codes

Hash ermöglicht es Ihnen, zwei verschiedene PIN-Codes einzurichten:

Normaler Code — Öffnet die App normal, mit allen Ihren Daten.
Vash-Code — Öffnet die App normal... aber alle Ihre Daten werden stillschweigend im Hintergrund gelöscht.

Was der Beobachter sieht

Die App öffnet sich normal. Keine Warnung, keine verdächtige Animation, keine Fehlermeldung. Der Bildschirm zeigt einfach eine leere App — als hätten Sie sie nie benutzt.

Der Beobachter denkt, Sie benutzen Hash nicht. In Wirklichkeit wurden alle Ihre Gespräche, Kontakte und Nachrichten unwiderruflich gelöscht in Sekundenbruchteilen.

Konfigurierbar

Sie wählen genau aus, was bei der Aktivierung des Vash-Modus gelöscht wird:

  • Ihre Kontakte
  • Ihre Nachrichten und Gespräche
  • Ihre privaten Notizen

Die Löschung erfolgt lokal UND auf dem Server. Wenn Sie zum Zeitpunkt der Aktivierung offline sind, wird die Server-Löschung in eine Warteschlange gestellt und ausgeführt, sobald die Verbindung wiederhergestellt ist.

Ihre Sprachanrufe sind Ende-zu-Ende-verschlüsselt über WebRTC (LiveKit). Die Stimme wird direkt zwischen den Geräten übertragen — der Server erleichtert nur den anfänglichen Verbindungsaufbau.

VERSCHLÜSSELTE VIDEOANRUFE

Gleiche Technologie wie bei Audioanrufen, plus Video. Jeder Stream wird einzeln verschlüsselt. Niemand kann Ihr Bild oder Ihren Ton abfangen.

FOTOS & VIDEOS

Jede Datei wird mit AES-256-GCM verschlüsselt, bevor sie Ihr Gerät verlässt. Der Server speichert nur unlesbare verschlüsselte Daten, die nach Empfang gelöscht werden.

DOKUMENTE

PDF, ZIP, jede Datei — das gleiche Verschlüsselungsniveau gilt. Der Dateiname, die Größe und der Inhalt werden alle verschlüsselt.
