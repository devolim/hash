Premiers pas

Téléchargez Hash gratuitement depuis l'App Store (iOS) ou Google Play (Android). L'application est gratuite et le restera toujours. Aucun achat intégré, aucun abonnement.
Non. Hash ne demande aucun numéro de téléphone, aucune adresse email et aucun mot de passe. Au premier lancement, vous créez un code PIN local et un Hash ID anonyme est généré automatiquement. C'est tout.
Votre Hash ID est un identifiant unique et anonyme attribué à votre appareil. Il n'est relié à aucune information personnelle. Personne ne peut remonter jusqu'à vous grâce à votre Hash ID.
Deux méthodes sont disponibles :

1. QR Code (recommandé) :L'un de vous affiche son QR code dans l'app, l'autre le scanne. La connexion est instantanée et anonyme.

2. Code temporaire : Hash génère un code à 6 chiffres valable 5 minutes. Partagez-le avec votre contact, et il le saisit dans l'app pour vous ajouter.
Hash fonctionne sur le principe "un appareil = un compte". Cela renforce votre sécurité : vos clés de chiffrement n'existent que sur un seul appareil. Vous pouvez cependant transférer votre compte vers un nouvel appareil via les réglages.

Messages

Oui. Dès que votre destinataire reçoit un message, celui-ci est immédiatement supprimé de nos serveurs. Si le destinataire ne se connecte pas dans les 24 heures, le message est automatiquement et définitivement détruit. Aucune copie n'est conservée.
Un média éphémère est une photo ou une vidéo que votre destinataire ne peut voir qu'une seule fois. Une fois visionnée, elle disparaît définitivement. Il est impossible de la revoir ou de la sauvegarder.
Vous pouvez envoyer des photos, vidéos, documents, messages vocaux et fichiers de tout type. Tous les fichiers sont chiffrés avec AES-256-GCM avant l'envoi et supprimés du serveur après réception.
Hash utilise le protocole Signal pour le chiffrement de bout en bout de vos messages. Chaque message est chiffré sur votre appareil avant d'être envoyé. Le serveur ne voit qu'un bloc de données illisible. Seul votre destinataire possède la clé pour le déchiffrer. De plus, le protocole Sealed Sender garantit que même le serveur ne sait pas qui envoie quoi à qui.
Oui. Hash vous permet de partager votre localisation de manière chiffrée. Votre position est chiffrée de la même façon que vos messages et supprimée du serveur après réception.

Appels

Oui. Tous les appels audio et vidéo sont chiffrés de bout en bout. La connexion se fait directement entre les appareils (peer-to-peer). Aucun enregistrement n'est réalisé. Aucune durée, aucun historique d'appel n'est conservé.
La qualité des appels dépend de votre connexion internet. Pour une expérience optimale :

• Utilisez une connexion Wi-Fi stable
• Assurez-vous que votre connexion 4G/5G est suffisante
• Fermez les autres applications qui consomment de la bande passante
• Vérifiez que les deux participants ont une bonne connexion
Oui. Hash prend en charge les appels audio et vidéo. Les deux sont chiffrés de bout en bout. Vous pouvez basculer entre l'audio et la vidéo pendant un appel.

Sécurité

Le Mode Vash est une fonctionnalité unique à Hash. Vous configurez un second code PIN. Si quelqu'un vous force à ouvrir l'application, vous entrez ce second code. L'app s'ouvre normalement mais toutes vos données sont silencieusement et irréversiblement supprimées : contacts, messages, conversations, fichiers. L'observateur voit une application vide. Aucune alerte n'est affichée.

Attention : cette action est irréversible. Toutes les données sont définitivement perdues.
Le bouton panique permet de supprimer instantanément et définitivement toutes les données de l'application en une seule action. Contrairement au Mode Vash, cette suppression est visible, l'utilisateur confirme explicitement la destruction de ses données.
Le verrouillage par secousse permet de verrouiller instantanément l'application en secouant votre téléphone. Utile si quelqu'un s'approche de vous pendant que vous utilisez Hash. Vous pouvez ajuster la sensibilité de détection dans les réglages.
Le Sealed Sender est un protocole qui empêche le serveur de savoir qui communique avec qui. Vos messages transitent de façon totalement anonyme : l'expéditeur, le destinataire et le contenu sont invisibles pour notre infrastructure. Même si quelqu'un intercepte le trafic, il ne peut pas déterminer qui envoie un message à qui.
Par conception, Hash ne stocke aucune information sur nos serveurs qui permettrait de réinitialiser votre PIN. Si vous avez configuré une phrase de récupération, vous pouvez l'utiliser pour récupérer l'accès. Sinon, les données de l'application sont irrémédiablement perdues et vous devrez réinstaller Hash pour repartir de zéro. C'est un choix de sécurité, pas un défaut.
Non. Le chiffrement de bout en bout signifie que seuls vous et votre destinataire pouvez lire vos messages. Le serveur ne voit que des données chiffrées qu'il est techniquement incapable de déchiffrer. De plus, le protocole Sealed Sender empêche le serveur de savoir qui communique avec qui. Même si nous recevions une injonction légale, nous serions dans l'impossibilité de fournir le contenu de vos conversations.

Notifications

Allez dans Réglages > Notifications et modifiez le nom affiché. Vous pouvez mettre "Rappel", "Notes", ou n'importe quel nom. Les notifications afficheront ce nom au lieu de "Hash", rendant l'application invisible au quotidien.
Dans Réglages > Notifications, activez l'option pour masquer le contenu. Les notifications afficheront un message générique au lieu du contenu réel de vos messages. Personne ne pourra lire vos conversations par-dessus votre épaule.
Vérifiez les points suivants :

• Permissions :Assurez-vous que Hash a l'autorisation d'envoyer des notifications (Réglages du téléphone > Hash > Notifications)
• Mode Ne pas déranger :Vérifiez qu'il n'est pas activé
• Batterie :Certains téléphones désactivent les notifications en mode économie d'énergie. Excluez Hash de l'optimisation batterie
• Connexion :Vérifiez votre connexion internet

Compte

Allez dans Réglages > Compte > Transférer mon compte. L'application génère un code de transfert. Saisissez ce code sur votre nouvel appareil. Le transfert déplace votre Hash ID et vos contacts vers le nouvel appareil. L'ancien appareil est automatiquement déconnecté.

Note : les messages ne sont pas transférés, conformément au principe de messages éphémères de Hash.
Allez dans Réglages > Compte > Supprimer mon compte. La suppression est immédiate et irréversible. Toutes vos données (Hash ID, contacts, clés de chiffrement, messages) sont définitivement effacées de votre appareil et de nos serveurs.

Vous pouvez également demander la suppression par email : hash-support@devolim.fr

Plus d'informations sur notre page de suppression de compte.
Non. La suppression d'un compte Hash est définitive et irréversible. Aucune donnée n'est conservée sur nos serveurs après la suppression. C'est un choix de conception pour garantir votre vie privée : si nous ne conservons rien, personne ne peut y accéder.
Aucune. Hash ne collecte strictement aucune donnée personnelle. Pas de numéro de téléphone, pas d'email, pas d'adresse IP, pas de métadonnées, pas d'analytics, pas de cookies, pas de traceurs. L'intégralité du code est open source et vérifiable par n'importe qui. Consultez notre politique de confidentialité pour plus de détails.

Dépannage

Vérifiez votre connexion internet. Les messages sont envoyés dès que la connexion est rétablie. Si le problème persiste, essayez de fermer et rouvrir l'application. Si votre contact a supprimé son compte ou vous a bloqué, le message ne pourra pas être délivré.
Essayez les étapes suivantes :

• Assurez-vous d'avoir la dernière version de Hash
• Redémarrez votre téléphone
• Vérifiez que votre système d'exploitation est à jour
• Si le problème persiste, contactez-nous à hash-support@devolim.fr en décrivant le problème
Envoyez un email à hash-support@devolim.fr avec une description du problème, votre modèle d'appareil et la version de votre système d'exploitation. Des captures d'écran ou une description détaillée des étapes pour reproduire le bug sont très utiles.
