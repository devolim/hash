2. 収集するデータ

Hash はデータ最小化の原則に基づいて設計されています。サービスの運用に必要な最小限のデータのみを収集します：

2.1 収集するデータ

  • Hash ID：ランダムに生成される一意の識別子（形式：000-000-XXX）で、実際の身元とは紐づいていません
  • デバイス識別子：アカウントをデバイスに関連付けるための一意の技術的識別子
  • プラットフォーム：使用しているデバイスの種類（iOS または Android）、通知の送信に必要
  • 公開暗号鍵：Signal Protocol 鍵束の公開鍵、連絡先が暗号化メッセージを送信するために必要
  • メールボックストークン：鍵から暗号的に導出されたトークン、メッセージの匿名受信に使用

2.2 収集しないデータ

  • メッセージの内容：すべてのメッセージはエンドツーエンド（E2E）で暗号化されています。たとえ望んでも、私たちはメッセージを読むことができません
  • 連絡先リスト：アドレス帳はデバイス上に保持され、私たちはアクセスしません
  • 位置情報：位置情報データは、ユーザーが自発的に位置情報メッセージを送信した場合にのみ送信されます。エンドツーエンドで暗号化されており、当社のサーバーで読み取ったり保存したりすることはできません
  • メールアドレス：不要であり、収集しません
  • 電話番号：不要であり、収集しません
  • IPアドレス：データベースに保存しません
  • 通信メタデータ：匿名トークンシステムにより、誰が誰と通信しているかを把握していません
  • 写真、ファイル、メディア：すべてのマルチメディアコンテンツは送信前にクライアント側で暗号化され、サーバー上では読み取り不可能な暗号化データとしてのみ転送されます

3. 暗号化とセキュリティ

Hash は最先端の暗号化技術を使用して通信を保護しています：

  • Signal Protocol：Signal や WhatsApp でも使用されているエンドツーエンド暗号化プロトコルで、前方秘匿性と将来秘匿性を提供します
  • AES-256：メッセージとファイルの暗号化に使用される軍事レベルの対称暗号化アルゴリズム
  • Curve25519：Diffie-Hellman 鍵交換のための楕円曲線
  • Double Ratchet：各メッセージは自動的に更新される一意のセッション鍵を使用します
  • Sealed Sender：サーバーがメッセージの送信者を知ることを防ぐ匿名送信者技術

お客様の秘密鍵はデバイスの外に出ることはありません。秘密鍵はローカルの暗号化ストレージ（Hive）に保存され、サーバーに送信されることはありません。

4. データの保存期間

  • 転送中のメッセージ：サーバー上に最大24時間保存されます。受信者が受信を確認すると、メッセージはサーバーから即座に不可逆的に削除されます
  • アカウントデータ：アカウントが有効な間保持されます。アカウント削除時（パニックボタンまたは手動削除）に即座かつ永久に削除されます
  • ファイルとメディア：メッセージと同じポリシーで、受信確認後に削除されます

5. ローカルストレージ

ローカルデータ（復号されたメッセージ、連絡先、鍵）は、暗号化された Hive データベースにお客様のデバイス上にのみ保存されます。これらのデータはサーバーやクラウドに同期されることはありません。

デバイスの紛失や機種変更の場合、これらのデータは永久に失われます。これは意図的な設計上の選択です：1つのデバイス = 1つのアカウント、復旧は不可能です。

6. 第三者へのデータ共有

データ共有はありません。Hash はアナリティクスツール、広告、トラッカーを一切使用していません。いかなる第三者にもデータを共有、販売、転送することはありません。

サーバーは Supabase がホスティングしています。これらのサーバーに保存されるデータは、読み取り不可能な暗号化データ（メッセージ）または最小限の技術データ（Hash ID、公開鍵）のいずれかです。

プッシュ通知は Firebase Cloud Messaging（Google LLC）を通じて配信されます。この目的で Google が処理するのは FCM デバイストークンのみです。メッセージの内容はこのサービスを通じて送信されません。Firebase プライバシーポリシー。

7. お客様の権利（GDPR）

一般データ保護規則（GDPR）に基づき、お客様には以下の権利があります：

  • アクセス権：お客様に関して保有するデータ（Hash ID、公開鍵、デバイス識別子）を請求できます
  • 訂正権：不正確なデータの修正を請求できます
  • 消去権：アプリ内のパニックボタンまたは設定セクションから、いつでもすべてのデータを削除できます。削除は即座かつ不可逆的です
  • データポータビリティ権：収集データが最小限であることを踏まえ、データのコピーを取得するためにお問い合わせいただけます
  • 異議申立権：アカウントを削除することでデータ処理に異議を申し立てることができます
  • 苦情申立権：CNIL（www.cnil.fr）に苦情を申し立てることができます

権利を行使するには、hash-privacy@devolim.fr までご連絡ください。

8. お客様の権利（米国プライバシー法）

カリフォルニア州（CCPA/CPRA）、バージニア州（VCDPA）、コロラド州（CPA）、またはプライバシー法を有する他の米国の州にお住まいの場合、以下の権利があります：

  • 知る権利：当社が収集する個人データおよびその使用方法に関する情報を請求できます
  • 削除する権利：アプリ内の削除機能または当社への連絡を通じて、いつでも個人データの削除を請求できます
  • 販売を拒否する権利：当社は、金銭的またはその他の有価の対価と引き換えに、お客様の個人データを第三者に販売、共有、または賃貸することはありません
  • 差別を受けない権利：プライバシー権を行使したことを理由に差別することはありません

これらの権利を行使するには、アプリ内のアカウント削除機能を使用するか、hash-privacy@devolim.fr までご連絡ください。

9. 最低年齢

Hash は13歳以上の方を対象としています。現地の法律で義務付けられている EU 加盟国では、最低年齢が16歳となる場合があります。

適用される最低年齢未満のお子様のデータを故意に収集することはありません。保護者の方で、お子様が同意なく Hash を使用していると思われる場合は、hash-privacy@devolim.fr までご連絡ください。

10. Cookie およびトラッキング技術

Hash アプリはCookie、トラッキングピクセル、アナリティクス SDK を一切使用しません。ウェブサイト hash.devolim.fr もトラッキング Cookie を使用していません。

11. 国際データ転送

最小限の技術データ（Hash ID、公開鍵）は、欧州経済領域（EEA）外のサーバーで処理される場合があります。ただし、すべての機密データはエンドツーエンドで暗号化されており、個人を特定できるデータを収集していないため、プライバシーへのリスクは最小限です。

12. データセキュリティ

以下のセキュリティ対策を実施しています：

  • すべての通信のエンドツーエンド暗号化（Signal Protocol）
  • ローカルデータの AES-256 暗号化
  • PIN コードおよび生体認証
  • 緊急消去のための破壊コード（デュレス PIN）
  • スクリーンショット防止（Android の FLAG_SECURE）
  • 配信後のサーバーからのメッセージ自動削除
  • セキュリティ定義関数（security definer）を介したデータベースへの排他的アクセス
  • すべてのテーブルに対する RLS（行レベルセキュリティ）ポリシー

13. 本ポリシーの変更

本プライバシーポリシーは随時更新される場合があります。重要な変更が行われた場合は、アプリ内通知でお知らせします。最終更新日はこのページの上部に記載されています。
