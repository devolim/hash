2. 수집하는 데이터

Hash는 데이터 최소화 원칙에 따라 설계되었습니다. 서비스 운영에 꼭 필요한 최소한의 정보만 수집합니다:

2.1 수집하는 데이터

  • Hash ID: 무작위로 생성되는 고유 식별자(형식 000-000-XXX)로, 실제 신원과 연결되지 않습니다
  • 기기 식별자: 계정을 기기에 연결하기 위한 고유 기술 식별자
  • 플랫폼: 사용하는 기기 유형(iOS 또는 Android), 알림 전송에 필요
  • 공개 암호화 키: Signal Protocol 키 묶음의 공개 키로, 연락처가 암호화된 메시지를 보내는 데 필요
  • 우편함 토큰: 키에서 암호학적으로 파생된 토큰으로, 익명 메시지 수신에 사용

2.2 수집하지 않는 데이터

  • 메시지 내용: 모든 메시지는 종단간 암호화(E2E)됩니다. 당사가 원하더라도 읽을 수 없습니다
  • 연락처 목록: 주소록은 기기에만 저장되며, 당사는 절대 접근하지 않습니다
  • 위치: 위치 데이터는 사용자가 자발적으로 위치 메시지를 보낼 때만 전송됩니다. 종단간 암호화되어 있으며 서버에서 읽거나 저장할 수 없습니다
  • 이메일 주소: 필요하지 않으며, 수집하지 않습니다
  • 전화번호: 필요하지 않으며, 수집하지 않습니다
  • IP 주소: 데이터베이스에 저장하지 않습니다
  • 통신 메타데이터: 익명 토큰 시스템 덕분에 누가 누구와 통신하는지 알 수 없습니다
  • 사진, 파일, 미디어: 모든 멀티미디어 콘텐츠는 전송 전 클라이언트 측에서 암호화되며, 서버에서는 읽을 수 없는 암호화된 데이터로만 전송됩니다

3. 암호화 및 보안

Hash는 최첨단 암호화 기술을 사용하여 통신을 보호합니다:

  • Signal Protocol: Signal과 WhatsApp에서도 사용되는 종단간 암호화 프로토콜로, 순방향 비밀성(forward secrecy)과 미래 비밀성(future secrecy)을 제공합니다
  • AES-256: 메시지 및 파일 암호화를 위한 군사 수준의 대칭 암호화 알고리즘
  • Curve25519: Diffie-Hellman 키 교환을 위한 타원 곡선
  • Double Ratchet: 각 메시지는 자동으로 갱신되는 고유한 세션 키를 사용합니다
  • Sealed Sender: 서버가 메시지 발신자를 알 수 없도록 하는 익명 발신자 기술

개인 키는 절대로 기기를 떠나지 않습니다. 개인 키는 암호화된 로컬 저장소(Hive)에 저장되며, 서버로 전송되지 않습니다.

4. 데이터 보관 기간

  • 전송 중인 메시지: 서버에 최대 24시간 저장됩니다. 수신자가 수신을 확인하면 메시지는 서버에서 즉시 영구적으로 삭제됩니다
  • 계정 데이터: 계정이 활성 상태인 동안 보관됩니다. 계정 삭제(비상 버튼 또는 수동 삭제) 시 즉시 영구적으로 삭제됩니다
  • 파일 및 미디어: 메시지와 동일한 정책으로, 수신 확인 즉시 삭제됩니다

5. 로컬 저장소

로컬 데이터(복호화된 메시지, 연락처, 키)는 암호화된 Hive 데이터베이스에 기기에서만 저장됩니다. 이 데이터는 서버나 클라우드에 동기화되지 않습니다.

기기 분실이나 교체 시 이 데이터는 영구적으로 소실됩니다. 이는 의도적인 설계입니다: 하나의 기기 = 하나의 계정, 복구 불가.

6. 제3자와의 데이터 공유

데이터 공유 없음. Hash는 분석 도구, 광고, 트래커를 사용하지 않습니다. 당사는 어떠한 제3자에게도 데이터를 공유, 판매 또는 이전하지 않습니다.

서버는 Supabase에서 호스팅됩니다. 서버에 저장되는 데이터는 읽을 수 없는 암호화된 데이터(메시지)이거나 최소한의 기술 데이터(Hash ID, 공개 키)입니다.

푸시 알림은 Firebase Cloud Messaging(Google LLC)을 통해 전달됩니다. 이 목적을 위해 Google에서 처리하는 것은 FCM 기기 토큰뿐입니다. 이 서비스를 통해 메시지 내용은 전송되지 않습니다. Firebase 개인정보 처리방침.

7. 귀하의 권리 (GDPR)

일반 데이터 보호 규정(GDPR)에 따라 귀하는 다음의 권리를 보유합니다:

  • 접근권: 당사가 보유한 귀하의 데이터(Hash ID, 공개 키, 기기 식별자)에 대한 정보를 요청할 수 있습니다
  • 정정권: 부정확한 데이터의 수정을 요청할 수 있습니다
  • 삭제권: 앱 내 비상 버튼이나 설정 섹션을 통해 언제든지 모든 데이터를 삭제할 수 있습니다. 삭제는 즉시 이루어지며 되돌릴 수 없습니다
  • 이동권: 수집되는 데이터의 양이 최소화되어 있으므로, 데이터 사본을 받으려면 당사에 연락하시기 바랍니다
  • 이의제기권: 계정을 삭제하여 데이터 처리에 이의를 제기할 수 있습니다
  • 불만 제기 권리: CNIL(www.cnil.fr)에 불만을 제기할 수 있습니다

권리를 행사하려면 hash-privacy@devolim.fr로 연락해 주세요.

8. 귀하의 권리 (미국 개인정보 보호법)

캘리포니아(CCPA/CPRA), 버지니아(VCDPA), 콜로라도(CPA) 또는 개인정보 보호 법률이 있는 다른 미국 주에 거주하는 경우, 다음과 같은 권리를 보유합니다:

  • 알 권리: 당사가 수집하는 개인 데이터 및 그 사용 방법에 대한 정보를 요청할 수 있습니다
  • 삭제 권리: 앱 내 삭제 기능 또는 당사에 연락하여 언제든지 개인 데이터의 삭제를 요청할 수 있습니다
  • 판매 거부 권리: 당사는 금전적 또는 기타 가치 있는 대가를 위해 개인 데이터를 제3자에게 판매, 공유 또는 임대하지 않습니다
  • 차별 금지 권리: 개인정보 보호 권리를 행사했다는 이유로 차별하지 않습니다

이러한 권리를 행사하려면 앱 내 계정 삭제 기능을 사용하거나 hash-privacy@devolim.fr로 연락해 주세요.

9. 최소 연령

Hash는 만 13세 이상을 대상으로 합니다. 현지 법률이 요구하는 일부 유럽연합 국가에서는 최소 연령이 16세일 수 있습니다.

당사는 해당 최소 연령 미만의 아동에 관한 데이터를 의도적으로 수집하지 않습니다. 부모 또는 보호자로서 자녀가 동의 없이 Hash를 사용하고 있다고 판단되는 경우, hash-privacy@devolim.fr로 연락해 주세요.

10. 쿠키 및 추적 기술

Hash 앱은 쿠키, 추적 픽셀, 분석 SDK를 일절 사용하지 않습니다. hash.devolim.fr 웹사이트도 추적 쿠키를 사용하지 않습니다.

11. 국제 데이터 이전

최소한의 기술 데이터(Hash ID, 공개 키)는 유럽 경제 지역(EEA) 외부에 위치한 서버에서 처리될 수 있습니다. 그러나 모든 민감한 데이터가 종단간 암호화되어 있고 개인 식별 정보를 수집하지 않으므로, 개인정보에 대한 위험은 최소화됩니다.

12. 데이터 보안

당사는 다음의 보안 조치를 시행합니다:

  • 모든 통신의 종단간 암호화 (Signal Protocol)
  • 로컬 데이터의 AES-256 암호화
  • PIN 코드 및 생체 인증
  • 긴급 삭제를 위한 위장 PIN (duress PIN)
  • 스크린샷 방지 (Android FLAG_SECURE)
  • 전달 후 서버에서 메시지 자동 삭제
  • 보안 함수(security definer)를 통한 데이터베이스 접근 제한
  • 모든 테이블에 RLS(Row Level Security) 정책 적용
