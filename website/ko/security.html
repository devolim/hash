흔적 제로.
약속이 아닌, 설계에 의해.

우리는 단순히 기존 메신저에 암호화를 추가한 것이 아닙니다. 당신의 데이터에 접근하는 것이 물리적으로 불가능하도록 모든 계층을 구축했습니다.

메시지의 여정

당신의 손가락에서
상대방의 화면까지.

Hash 메시지의 여정을 단계별로 따라가 보세요. 왜 누구도 — 우리조차도 — 당신의 대화를 읽을 수 없는지 이해하게 될 것입니다.

왼쪽에서 당신이 보는 것을 볼 수 있습니다: 간단한 "안녕". 하지만 서버가 받는 것은 전혀 읽을 수 없는 610바이트의 암호화된 데이터입니다.

이중 암호화 계층. 매 메시지마다 고유한 임시 키. 누군가 모든 것을 가로채더라도 — 수신자의 개인 키 없이는 무용지물이며, 그 키는 오직 수신자의 휴대폰에만 저장되어 있습니다.

1

메시지를 입력합니다

앱에서 "안녕!"이라고 입력합니다. 이 단계에서 메시지는 휴대폰 메모리에만 존재합니다. 아직 아무것도 전송되지 않았습니다.

2

Signal Protocol로 암호화

"보내기"를 누르는 순간, 메시지는 Signal Protocol에 의해 해독 불가능한 문자열로 변환됩니다.

쉽게 말해: 메시지가 디지털 금고에 잠기고, 오직 상대방만이 그 키를 가지고 있습니다. 누군가 전송 중 메시지를 가로채더라도 의미 없는 문자와 숫자의 나열만 보게 됩니다.

DOUBLE RATCHET CURVE25519 AES-256
3

Sealed Sender: 보이지 않는 봉투

우편으로 편지를 보내되 봉투에 발신자 주소가 없는 것을 상상해 보세요. 이것이 바로 Hash의 "Sealed Sender" 시스템이 하는 일입니다.

암호화된 메시지는 서버의 익명 우편함에 투입됩니다. 서버는 누가 메시지를 넣었는지 모릅니다. 어떤 우편함에 넣어야 하는지만 알 뿐 — 그 우편함 소유자의 신원은 알지 못합니다.

4

서버 통과

서버는 단순한 맹인 우체부 역할을 합니다. 서버가 보는 것은 다음과 같습니다:

서버가 아는 것: "누군가가 #A7X9 우편함에 무언가를 넣었다."
서버가 모르는 것: 누가 보냈는지. 무엇인지. 실제로 누구에게 보내는 것인지.

서버는 어떤 메타데이터도 저장하지 않습니다: IP 주소 없음, 전송 타임스탬프 없음, 발신자와 수신자 간의 연결 없음.

5

상대방이 메시지를 받습니다

상대방의 휴대폰이 익명 우편함의 내용을 가져옵니다. 기기를 떠난 적이 없는 개인 키로 로컬에서 메시지를 복호화합니다.

"안녕!"이 화면에 나타납니다. 임무 완료.

6

메시지가 서버에서 사라집니다

상대방이 수신을 확인하는 즉시, 서버는 메시지를 영구 삭제합니다. 휴지통 없음, 아카이브 없음, 백업 없음. 데이터베이스에서 데이터가 지워집니다.

7

로컬 만료

상대방의 기기에서 메시지는 설정한 시간에 따라 자동 소멸됩니다: 읽은 직후, 5분, 1시간... 당신이 결정합니다.

결과적으로: 서버에 흔적 제로, 기기에 흔적 제로. 메시지는 읽히는 시간만큼만 존재하고, 영원히 사라집니다.

서버는 메시지를 읽을 수도, 발신자를 식별할 수도, 흔적을 보관할 수도 없습니다. 메시지는 수신 즉시 삭제됩니다.

아키텍처

5중 보호 계층.

Hash는 하나의 기술에만 의존하지 않습니다. 각 계층이 서로를 강화하여, 하나의 요소가 침해되더라도 데이터가 위험에 처하지 않는 시스템을 만듭니다.

1

종단간 암호화 (SIGNAL PROTOCOL)

모든 메시지는 "Double Ratchet" 알고리즘을 통해 고유한 키로 암호화됩니다. 쉽게 말해: 누군가 한 메시지를 복호화하더라도 다음 메시지는 복호화할 수 없습니다. 각 메시지에는 고유한 자물쇠가 있습니다.

파일(사진, 동영상, 문서)의 경우, Hash는 추가로 AES-256-GCM 암호화를 사용합니다. 파일은 휴대폰을 떠나기 전에 암호화됩니다.

2

SEALED SENDER (네트워크 익명성)

일반 메신저는 신원이 첨부된 채 메시지를 보냅니다 — 봉투에 이름을 쓰는 것과 같습니다. Hash는 "익명 우편함" 시스템을 사용합니다: 서버는 누가 보냈는지 알지 못한 채 메시지를 전달합니다.

결과: 서버가 침해되더라도 공격자는 누가 누구와 대화하는지 재구성할 수 없습니다.

3

자동 삭제

메시지는 수신 확인 즉시 서버에서 삭제됩니다. 메시지가 수신되지 않더라도 24시간 후 자동으로 파기됩니다.

휴대폰에서 메시지는 설정한 시간에 따라 자동 소멸됩니다: 즉시, 5분, 15분, 30분, 1시간, 3시간, 6시간 또는 12시간.

4

로컬 접근 보호

앱은 6자리 PIN 코드 및/또는 생체인증(Face ID, 지문)으로 보호됩니다. 실패 횟수가 많아지면 지수적 지연과 함께 앱이 잠깁니다.

5

잠긴 데이터베이스

서버 측에서 어떤 사용자도 데이터베이스에 직접 쓸 수 없습니다. 모든 작업(메시지 전송, 연락처 추가, 계정 삭제)은 실행 전 각 작업을 검증하는 보안 함수를 통과합니다.

은행 창구와 같습니다: 직접 금고를 만지지 않습니다. 요청을 하면 시스템이 권한을 확인한 후 실행합니다.

VASH 모드.
세계 유일.

세계 어떤 메신저에도 존재하지 않는 긴급 보안 시스템.

Hash는 파괴 코드를 제공하는 유일한 메신저입니다. 겉보기에는 정상적으로 앱을 여는 두 번째 PIN 코드 — 하지만 실제로는 백그라운드에서 조용히 모든 데이터를 삭제합니다.

대화, 연락처, 암호화 키, 파일 — 모든 것이 순식간에 파괴됩니다. 관찰자는 빈 앱을 보고 사용하지 않는다고 생각합니다. 흔적 없음. 알림 없음. 돌이킬 수 없음.

Signal은 이 기능을 거부했습니다. Telegram은 고려한 적도 없습니다. WhatsApp은 생각조차 하지 않았습니다. 오직 Hash만이 구현했습니다.

시나리오

상상해 보세요: 누군가가 당신에게 휴대폰 잠금을 해제하라고 요구합니다. 검문, 압박 상황, 악의적인 사람. 거절할 수 없습니다. 어떻게 하시겠습니까?

해결책: 두 개의 PIN 코드

Hash는 서로 다른 두 개의 PIN 코드를 설정할 수 있습니다:

일반 코드 — 모든 데이터와 함께 정상적으로 앱을 엽니다.
Vash 코드 — 정상적으로 앱을 엽니다... 하지만 모든 데이터가 백그라운드에서 조용히 삭제됩니다.

관찰자가 보는 것

앱이 정상적으로 열립니다. 알림 없음, 수상한 애니메이션 없음, 오류 메시지 없음. 화면에는 빈 앱만 표시됩니다 — 마치 한 번도 사용한 적 없는 것처럼.

관찰자는 Hash를 사용하지 않는다고 생각합니다. 실제로는 모든 대화, 연락처, 메시지가 순식간에 돌이킬 수 없이 삭제되었습니다.

설정 가능

Vash 모드 활성화 시 삭제할 항목을 정확히 선택할 수 있습니다:

  • 연락처
  • 메시지 및 대화
  • 개인 메모

삭제는 로컬 및 서버 모두에서 수행됩니다. 활성화 시 오프라인 상태라면, 서버 삭제는 대기열에 추가되어 연결이 복구되는 즉시 실행됩니다.

통화 및 파일

모든 것이 암호화됩니다. 절대적으로 모든 것.

암호화 음성 통화

음성 통화는 WebRTC(LiveKit)를 통해 종단간 암호화됩니다. 음성은 기기 간 직접 전송됩니다 — 서버는 초기 연결만 중개합니다.

암호화 영상 통화

음성 통화와 동일한 기술에 영상이 추가됩니다. 각 스트림이 개별적으로 암호화됩니다. 누구도 당신의 영상이나 음성을 가로챌 수 없습니다.

사진 및 동영상

모든 파일은 기기를 떠나기 전 AES-256-GCM으로 암호화됩니다. 서버는 읽을 수 없는 암호화된 데이터만 저장하며, 수신 후 삭제됩니다.

문서

PDF, ZIP, 모든 파일 — 동일한 수준의 암호화가 적용됩니다. 파일 이름, 크기, 내용 모두 암호화됩니다.

데이터

우리가 수집하지 않는 것.

데이터를 보호하는 가장 좋은 방법은 수집하지 않는 것입니다.

절대 수집하지 않음

  • 메시지 내용
  • 연락처 목록
  • 전화번호
  • 이메일 주소
  • IP 주소
  • 위치 정보
  • 메타데이터 (누가 누구와 대화하는지)
  • 통화 기록
  • 주소록
  • 광고 식별자

일시적 저장

  • 익명 Hash ID (고유 식별자)
  • 공개 암호화 키
  • 전송 중 암호화된 메시지 (최대 24시간)

이러한 최소한의 데이터조차도 당신을 식별할 수 없습니다. Hash ID는 어떤 개인정보와도 연결되어 있지 않습니다.
