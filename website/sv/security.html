Security

Zero trace.
By design, not by promise.

We didn't simply add encryption to a traditional messaging app. We built every layer so that it is physically impossible to access your data.

THE JOURNEY OF A MESSAGE

FROM YOUR FINGER TO
YOUR CONTACT'S SCREEN.

Follow the journey of a Hash message, step by step. You'll understand why no one — not even us — can read your conversations.

On the left, you see what you see: a simple "Hi". But what the server receives is a sequence of 610 bytes of totally unreadable encrypted data.

Two layers of encryption. A unique ephemeral key for each message. Even if someone intercepts everything — it's useless without the recipient's private key, stored only on their phone.

1

You type your message

You write "Hi!" in the app. At this point, the message only exists in your phone's memory. Nothing has been sent yet.

2

Encryption with the Signal Protocol

As soon as you press "Send", your message is transformed into a string of incomprehensible characters using the Signal Protocol.

In simple terms: your message is locked in a digital safe that only your contact has the key to. Even if someone intercepts the message in transit, they'll only see a meaningless string of letters and numbers.

DOUBLE RATCHET CURVE25519 AES-256
3

Sealed Sender: the invisible envelope

Imagine sending a letter by mail, but with no return address on the envelope. That's exactly what Hash's "Sealed Sender" system does.

The encrypted message is dropped into an anonymous mailbox on our server. The server doesn't know who dropped the message. It only knows which mailbox to put it in — but it doesn't know the identity of the mailbox owner.

4

Passing through the server

The server acts as a simple blind postman. Here's what it sees:

What the server knows: "Someone dropped something in mailbox #A7X9."
What the server does NOT know: Who sent it. What it is. Who it's really for.

The server stores no metadata: no IP address, no send timestamp, no link between sender and recipient.

5

Your contact receives the message

Your contact's phone retrieves the contents of their anonymous mailbox. It decrypts the message locally with their private key — which has never left their device.

"Hi!" appears on their screen. Mission accomplished.

6

The message disappears from the server

As soon as your contact confirms receipt, the server permanently deletes the message. No recycle bin, no archive, no backup. The data is erased from the database.

7

Local expiration

On your contact's device, the message self-destructs according to the duration you chose: immediately after reading, 5 minutes, 1 hour... you decide.

In the end: zero trace on the server, zero trace on devices. The message existed just long enough to be read, then it vanished forever.

Architecture

5 LAYERS OF PROTECTION.

Hash doesn't rely on a single technology. Each layer reinforces the others, creating a system where even compromising one element doesn't endanger your data.

1

END-TO-END ENCRYPTION (SIGNAL PROTOCOL)

Each message is encrypted with a unique key using the "Double Ratchet" algorithm. In simple terms: even if someone decrypts one message, they won't be able to decrypt the next one. Each message has its own lock.

For files (photos, videos, documents), Hash uses additional AES-256-GCM encryption. The file is encrypted before leaving your phone.

2

SEALED SENDER (NETWORK ANONYMITY)

Traditional messaging apps send your messages with your identity attached — it's like writing your name on the envelope. Hash uses an "anonymous mailbox" system: the server delivers the message without knowing who sent it.

Result: even if the server is compromised, an attacker cannot reconstruct who is talking to whom.

3

AUTOMATIC DELETION

Messages are deleted from the server as soon as receipt is confirmed. Even if a message is never retrieved, it is automatically destroyed after 24 hours.

On your phone, messages self-destruct according to the duration you choose: immediately, 5 min, 15 min, 30 min, 1h, 3h, 6h, or 12h.

4

LOCAL ACCESS PROTECTION

The app is protected by a 6-digit PIN code and/or biometrics (Face ID, fingerprint). After too many failed attempts, the app locks with an exponential delay.

5

LOCKED DATABASE

On the server side, no user can write directly to the database. All operations (sending a message, adding a contact, deleting an account) go through secure functions that verify each action before executing it.

It's like a bank counter: you never touch the vault yourself. You make a request, and the system verifies you have permission before acting.

VASH MODE.
WORLD FIRST.

AN EMERGENCY SECURITY SYSTEM THAT EXISTS IN NO OTHER MESSAGING APP IN THE WORLD.

Hash is the only messaging app to offer a destruction code. A second PIN code that, on the surface, opens the app normally — but in reality silently erases all your data in the background.

Conversations, contacts, encryption keys, files — everything is destroyed in a fraction of a second. The observer sees an empty app and thinks you don't use it. No trace. No alert. Irreversible.

Signal rejected this feature. Telegram never considered it. WhatsApp never even thought of it. Only Hash has implemented it.

The scenario

Imagine: someone asks you to unlock your phone. A checkpoint, a pressure situation, a malicious person. You can't refuse. What do you do?

The solution: two PIN codes

Hash allows you to set up two different PIN codes:

Normal code — Opens the app normally, with all your data.
Vash code — Opens the app normally... but all your data is silently deleted in the background.

What the observer sees

The app opens normally. No alert, no suspicious animation, no error message. The screen simply shows an empty app — as if you had never used it.

The observer thinks you don't use Hash. In reality, all your conversations, contacts, and messages have been irreversibly deleted in a fraction of a second.

Configurable

You choose exactly what gets deleted when Vash mode is activated:

  • Your contacts
  • Your messages and conversations
  • Your private notes

Deletion is performed locally AND on the server. If you are offline at the time of activation, the server deletion is queued and executed as soon as the connection returns.

Calls & Files

EVERYTHING IS ENCRYPTED. ABSOLUTELY EVERYTHING.

ENCRYPTED AUDIO CALLS

Your voice calls are end-to-end encrypted via WebRTC (LiveKit). Voice is transmitted directly between devices — the server only facilitates the initial connection.

ENCRYPTED VIDEO CALLS

Same technology as audio calls, with video added. Each stream is individually encrypted. No one can intercept your image or sound.

PHOTOS & VIDEOS

Each file is encrypted with AES-256-GCM before leaving your device. The server only stores unreadable encrypted data, deleted after receipt.

DOCUMENTS

PDF, ZIP, any file — the same level of encryption applies. The file name, its size, and its content are all encrypted.
