安全

零痕迹。
源于设计，而非承诺。

我们不是简单地给普通通讯应用加上加密。 我们构建了每一层架构，使得物理上不可能访问您的数据。

消息的旅程

从您的指尖到
联系人的屏幕。

跟随一条 Hash 消息的旅程，逐步了解。您将理解为什么 没有人 — 甚至是我们 — 能读取您的对话。

左侧是您看到的：一条简单的"你好"。 但服务器收到的是一串完全不可读的 610 字节加密数据。

两层加密。每条消息都有一个唯一的临时密钥。 即使有人截获了所有内容 — 没有接收者的私钥也毫无用处， 而私钥仅存储在接收者的手机上。

1

您输入消息

您在应用中写下"你好！"。此时，消息仅存在于您手机的内存中。什么都还没有发送。

2

使用 Signal Protocol 加密

当您按下"发送"时，您的消息通过 Signal Protocol 被转换为一串无法理解的字符。

简单来说：您的消息被锁在一个数字保险箱中，只有您的联系人拥有钥匙。即使有人在传输途中截获消息，他们看到的也只是毫无意义的字母和数字。

3

Sealed Sender：隐形信封

想象您寄一封信，但信封上没有寄件人地址。 这正是 Hash 的 "Sealed Sender" 系统所做的。

加密后的消息被放入我们服务器上的匿名信箱中。 服务器不知道是谁放入了消息。它只知道放入哪个信箱 — 但不知道信箱主人的身份。

4

经过服务器

服务器扮演一个盲人邮递员的角色。以下是它所看到的：

服务器知道的："有人在 #A7X9 信箱中放了什么东西。"
服务器不知道的：谁发的。是什么。真正要发给谁。

服务器不存储任何元数据：没有 IP 地址，没有发送时间戳，没有发送者和接收者之间的关联。

5

您的联系人收到消息

您联系人的手机从其匿名信箱中获取内容。 消息在本地使用其私钥解密 — 私钥从未离开过其设备。

"你好！"出现在其屏幕上。任务完成。

6

消息从服务器消失

一旦您的联系人确认收到，服务器永久删除该消息。 没有回收站，没有归档，没有备份。数据从数据库中被抹除。

7

本地过期

在您联系人的设备上，消息根据您选择的时间自动销毁： 阅读后立即、5分钟、1小时...由您决定。

最终：服务器上零痕迹，设备上零痕迹。消息存在的时间仅够被阅读，然后永远消失。

架构

5 层保护。

Hash 不依赖单一技术。每一层都强化其他层， 创建一个即使某个元素被攻破也不会危及您数据的系统。

1

端到端加密 (SIGNAL PROTOCOL)

每条消息都使用 "Double Ratchet" 算法生成的唯一密钥进行加密。 简单来说：即使有人破解了一条消息，也无法破解下一条。 每条消息都有自己的锁。

对于文件（照片、视频、文档）， Hash 使用额外的 AES-256-GCM 加密。文件在离开您的手机之前就已加密。

2

SEALED SENDER（网络匿名）

传统通讯应用发送消息时会附上您的身份 — 就像在信封上写上您的名字。 Hash 使用"匿名信箱"系统：服务器投递消息时不知道是谁发送的。

结果：即使服务器被攻破，攻击者也无法重建谁在和谁通信。

3

自动删除

消息在确认接收后立即从服务器删除。 即使消息从未被取回，也会在24小时后自动销毁。

在您的手机上，消息根据您选择的时长自动销毁： 立即、5分钟、15分钟、30分钟、1小时、3小时、6小时或12小时。

4

本地访问保护

应用受6位数 PIN 码和/或生物识别（Face ID、指纹）保护。 多次尝试失败后，应用将以指数级延迟锁定。

5

锁定的数据库

在服务器端，没有用户可以直接写入数据库。 所有操作（发送消息、添加联系人、删除账户）都通过安全函数进行， 在执行前验证每个操作。

就像银行柜台：您永远不会亲自触碰保险库。 您提出请求，系统在执行前验证您是否有权限。

VASH 模式。
全球独创。

一个在全球任何其他通讯应用中都不存在的紧急安全系统。

Hash 是唯一提供销毁码的通讯应用。 第二个 PIN 码，表面上正常打开应用 — 但实际上 在后台静默清除您的所有数据。

对话、联系人、加密密钥、文件 — 一切在瞬间被销毁。 旁观者看到一个空的应用，以为您不使用它。 无痕迹。无警告。不可逆。

Signal 拒绝了这个功能。Telegram 从未考虑过。WhatsApp 甚至没有想到。 只有 Hash 实现了它。

使用场景

想象：有人要求您解锁手机。 一次检查、一个压力场景、一个恶意的人。 您无法拒绝。怎么办？

解决方案：两个 PIN 码

Hash 允许您设置两个不同的 PIN 码：

正常码 — 正常打开应用，显示所有数据。
Vash 码 — 正常打开应用...但所有数据在后台被静默删除。

旁观者看到的

应用正常打开。没有警告，没有可疑动画，没有错误消息。 屏幕只是显示一个空的应用 — 就像您从未使用过一样。

旁观者以为您不使用 Hash。实际上，您的所有对话、联系人和消息已在一瞬间被 不可逆地删除。

可配置

您可以精确选择激活 Vash 模式时删除的内容：

  • 您的联系人
  • 您的消息和对话
  • 您的私密笔记

删除在本地和服务器上同时执行。如果激活时您处于离线状态， 服务器端的删除会被加入队列，并在连接恢复后立即执行。

通话与文件

一切都被加密。绝对一切。

加密语音通话

您的语音通话通过 WebRTC (LiveKit) 端到端加密。 声音直接在设备之间传输 — 服务器仅促进初始连接。

加密视频通话

与语音通话相同的技术，加上视频。 每个流都单独加密。没有人可以拦截您的画面或声音。

照片和视频

每个文件在离开您的设备前都使用 AES-256-GCM 加密。 服务器仅存储不可读的加密数据，接收后即删除。

文档

PDF、ZIP，任何文件 — 应用相同级别的加密。 文件名、大小和内容全部被加密。

数据

我们不收集的内容。

保护您数据的最佳方式就是不收集它们。

从不收集

  • 消息内容
  • 联系人列表
  • 手机号码
  • 电子邮箱
  • IP 地址
  • 位置信息
  • 元数据（谁和谁通信）
  • 通话记录
  • 通讯录
  • 广告标识符

临时存储

  • 匿名 Hash ID（唯一标识符）
  • 加密公钥
  • 传输中的加密消息（最长24小时）

即使是这些最少的数据也无法识别您的身份。 您的 Hash ID 不与任何个人信息关联。
